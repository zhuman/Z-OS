Queues

Queues provide a means of message/data passing between threads in a FIFO (First In First Out) manner. Queues are only thread-safe with one thread receiving the data. If more than one receiver tries to read data off of the queue, the results could be highly unpredictable. Therefore, if more than one receiving thread is wanted, users should wrap the queue accesses in a semaphore/mutex.
