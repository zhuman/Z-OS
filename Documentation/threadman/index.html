The Thread Manager

The thread manager is Z-OS's second most important subsystem, after the Object Manager. It manages all user and system threads and ensures that threads of equal priority receive equal processing time. It also interacts with synchronization primitives and other objects that implement the IWaitable interface to allow threads to wait for objects. Because of this interaction, the wait methods have been placed under the thread manager heading as well as under the object manager heading.

The thread manager also provides a few static methods for low-level synchronizations, interrupts, and timing. They are:

  • EnterCriticalSection
  • ExitCriticalSection
  • GetSystemTime
  • WaitForObject
  • WaitForObjects
  • FinishWait
