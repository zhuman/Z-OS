The IGeneric Interface

The IGeneric interface is mostly only a concern of those wishing to create their own objects, but it also is a good idea to understand how the object manager initializes and destroys objects. When CreateObject is called, the object manager first sets up its internal representation of the object. After this is done, it calls GetInterface in much the same way that user code would. The interface it asks for is IGeneric. If this interface is unimplemented by the object, it simply returns the new handle. If it is implemented, it calls the Create method of the interface, passing a pointer to its internal representation of the object, an InternalObject structure.

When an object is no longer in use by the system, the object manager will call the Destroy method of this interface before finalizing its destruction. This method should deallocate any memory that was allocated and remove any references to the object from the type manager's private structures.
