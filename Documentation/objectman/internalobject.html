The InternalObject Structure

The object manager internally stores InternalObject structures as its means of representing all of the objects on the system. This structure is documented here because of its use to type managers. It is the structure passed to the Create and Destroy function of the IGeneric interface.

Definition

typedef struct
{
	UInt16 Type;
	UInt16 DataLength;
	UInt16 Flags;
	UInt16 OpenHandles;
	UInt16 KernelReferences;
	char* Name;
	void* Data;
} InternalObject;

Fields

Type

This stores the type code for the object.

DataLength

This can be used by type managers, specifically for storing a length value pertaining to the Data pointer (see below).

Flags

This stores internal object flags.

OpenHandles

This stores the number of open handles the object has.

KernelReferences

Various subsystems reference objects. They may increment and decrement this field to show this.

Name

This is the null-terminated name string for the object or null if it has no name.

Data

This field should be used by type managers to store data or pointers to structures containing data about the object. It is never touched by the object manager, so type managers are responsible for memory allocation and deallocation. The DataLength field is provided as another way to store data.
