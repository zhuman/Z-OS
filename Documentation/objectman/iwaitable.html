The IWaitable Interface

An important aspect of the thread manager is the ability of threads to wait upon objects. This is accomplished using the IWaitable interface. This is mostly only of concern to users who wish to create and manage their own object types. Any object implementing this interface is able to be waited upon by WaitForObject and the other wait functions.

Only one function is required by the IWaitable interface, called StartWait. It is called internally by the wait functions and is used to register a wait with the type managers if needed. It also allows type managers to signal that no wait is required by the thread.
